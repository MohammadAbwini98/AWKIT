---

## Objective

Remove dummy reports from the Reports page.

Reports page must show only real reports generated by real workflow or instance executions.

---

## Current Problem

Reports page may show sample/demo/dummy reports. This is misleading.

The user wants:

```text
wipe the dummy reports
only show real reports
```

---

## Required Behavior

On first app start or when no real reports exist:

```text
Reports page shows empty state.
No fake reports appear.
```

Empty state:

```text
No reports yet.
Run a workflow to generate your first execution report.
```

When a real workflow run completes:

```text
Report is generated.
Report appears in Reports page.
Report can be opened/viewed.
```

## Wipe Dummy Reports

If dummy reports were already persisted into userData by previous versions, add cleanup logic carefully.

Recommended:

```text
Only remove records explicitly marked as demo/sample/mock.
Do not delete real user reports.
```

Add marker detection if available:

```json
{
  "source": "demo"
}
```

If no marker exists, do not blindly delete all reports. Instead stop seeding future dummy reports.
